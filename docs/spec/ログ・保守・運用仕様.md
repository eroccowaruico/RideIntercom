# RideIntercom ログ・保守・運用仕様

## 目的

本仕様は、通話継続を最優先にしつつ、codecフォールバックとメタデータ不整合を運用時に追跡可能にするための診断ログ要件を定義する。

## 基本方針

| 項目 | 方針 |
|---|---|
| 通信継続 | フォールバックやメタデータ不整合が発生しても通信を継続する |
| アプリ安定性 | ログ記録失敗を理由に例外送出・クラッシュを起こさない |
| 観測性 | Diagnostics画面とOSログ（Unified Logging）で同じ事象を追跡できるようにする |
| 収集性 | リリース後も `log show` で取得できる形式で出力する |

## 監視対象イベント

| イベント種別 | 発火条件 | 継続動作 |
|---|---|---|
| TX codec fallback | requestedCodec != encodedCodec または fallbackReason != null | 送信継続 |
| RX metadata mismatch | encodedVoice.codec と transmitMetadata.encodedCodec が不一致 | 受信処理継続 |
| encoder empty payload | HE-AACで空payloadとなりkeepalive化 | keepalive送信継続 |
| encoding failed | エンコード例外発生時にkeepalive化 | keepalive送信継続 |

## Diagnostics表示要件

| 表示箇所 | 項目 | 表示内容 |
|---|---|---|
| Diagnostics > Live Status | Codec Safety Summary | `TX FB #n / ...` と `RX META #n / ...` を1行で表示 |
| 集計値 | transmitFallbackCount | 送信フォールバック累積件数 |
| 集計値 | receiveMetadataMismatchCount | 受信メタデータ不整合累積件数 |
| 最新事象 | lastTransmitFallbackSummary | 最新フォールバックの要約 |
| 最新事象 | lastReceiveMetadataMismatchSummary | 最新メタデータ不整合の要約 |

## リリースログ出力要件

| 項目 | 値 |
|---|---|
| ログ基盤 | OSLog / Unified Logging |
| subsystem | com.yowamushi-inc.RideIntercom |
| category | codec-diagnostics |
| レベル | error |

| ログイベント | 必須フィールド |
|---|---|
| tx fallback | route, streamID, sequenceNumber, requestedCodec, encodedCodec, fallbackReason |
| rx metadata mismatch | peerID, streamID, sequenceNumber, metadataCodec, actualCodec |

## 取得手順（運用）

| 目的 | コマンド例 |
|---|---|
| 最近1時間のcodec診断ログ取得 | `log show --last 1h --predicate 'subsystem == "com.yowamushi-inc.RideIntercom" AND category == "codec-diagnostics"'` |
| リアルタイム監視 | `log stream --predicate 'subsystem == "com.yowamushi-inc.RideIntercom" AND category == "codec-diagnostics"'` |

## 保守要件

| 項目 | 要件 |
|---|---|
| 後方互換 | fallbackReason列挙値は既存値を維持し、追加時は受信側で未知値に耐性を持たせる |
| 障害解析 | Diagnostics表示とOSログのイベント名・意味を揃える |
| テスト | fallback検知とmetadata mismatch検知をユニットテストで固定する |
